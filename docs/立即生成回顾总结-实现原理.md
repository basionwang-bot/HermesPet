# 立即生成回顾总结 —— 实现原理

> **它在哪**：菜单栏 / 灵动岛菜单里的「🌱 立即生成总结回顾」
> **一句话**：不管时间到没到，立刻让桌宠把你**最近这一阵子**（约两周）的状态攒起来，做一份带感情、也带实用建议的大回顾。
> **对应代码**：`PeriodicReviewService.generateNow()`

---

## 一、先分清两个容易混的功能

HermesPet 里有**两层**「回顾」，时间尺度不一样，别搞混：

| | 📰 今日早报 | 🌱 总结回顾（本文） |
|---|---|---|
| 看的时间跨度 | **昨天**（一天） | **这一阵子**（约 1-2 周，甚至更长） |
| 什么时候出现 | 每天第一次打开 app 自动弹 | 攒够一段时间自动弹 / 你手动点「立即生成」 |
| 代码 | `MorningBriefingService` | `PeriodicReviewService` |
| 数据来源 | 当天的原始活动记录 | **不查原始数据**，只读早报每天攒下来的"日报归档" |

**关键关系**：今日早报是"砖"，总结回顾是"墙"。
早报每天帮你回顾完昨天后，会偷偷把那天的要点**存档**一份（叫 `daily_journal`，一天一行）。总结回顾要做的时候，**不重新去翻你的活动记录**，而是直接读这些存好的存档，省事又快。

```
每天的活动  ──早报每天提炼──▶  daily_journal（一天一行的存档）
                                      │
                            总结回顾 直接读这些存档
                                      │
                                      ▼
                          一份跨越多天的"这阵子"大回顾
```

## 二、数据是从哪来的（上游三层）

要看懂总结回顾，得先知道这些"存档"里的料怎么攒出来的。整条链是这样的：

```
【第 0 层：默默采集】 ActivityRecorder / UserIntentRecorder
   · 你切换了哪些 app、用了多久（NSWorkspace 监听）
   · 键盘鼠标的活跃节奏（只记"有没有在动"，不记打了什么字）
   · 开了「意图感知」才有：当前窗口标题、碰过的文档路径
   · 你问 AI 的话题（推断你在关注什么）
        │  全部存进本地 SQLite 数据库（ActivityStore）
        ▼
【第 1 层：每日早报】 MorningBriefingService（每天一次）
   · 拉昨天的活动 → 让 AI 写成一段温暖回顾
   · 同时把当天要点压缩成一小段结构化数据存档：
        最常用的 app、关注的窗口、碰过的文档、聊了几个话题
        （这一行就叫 daily_journal）
        ▼
【第 2 层：总结回顾】 PeriodicReviewService ← 你点的「立即生成」
   · 读最近 N 天的 daily_journal 存档
   · 让 AI 从中提炼出"规律"和"故事感"，写成大回顾
```

> **隐私说明（重要）**：第 0、1 层采集的所有数据都存在你**本机**的 SQLite 数据库里，平时**不上传任何地方**。只有在生成回顾的那一刻，才会把提炼后的要点（app 名、文档名、话题等）发给你**自己选定的那个 AI 后端**去写文字。所以"用哪个 AI 写早报/回顾"是让你**显式选**的（设置里的 `morningBriefingBackend`），就是为了让你清楚这些数据交给了谁。

## 三、点一下「立即生成总结回顾」之后，发生了什么

```
你点「🌱 立即生成总结回顾」
        │
        ▼
1. generateNow() → 标记为「手动」模式（不管周期到没到，直接做）
        │
        ▼
2. 读最近 14 天的 daily_journal 存档
        │
        ├─ 存档不够 3 天？→ 友好提示「日报攒得还不够（目前 X 天），
        │                      再用几天让我多了解你一点」→ 结束
        │
        ▼
3. 把这些存档拼成一段 prompt（角色设定 + 要给的两样价值 +
   你选的语气风格 + 每天一行的要点数据）
        │
        ▼
4. 灵动岛立刻进入"工作中"脉冲 + 横幅文字「🌱 正在整理这阵子…」
   （让你点完不是干等，马上有反馈）
        │
        ▼
5. 把 prompt 发给你选定的 AI 后端，流式拿回回顾正文
        │
        ▼
6. 灵动岛打个对勾 ✅ → 自动新建一个对话「🌱 这阵子的回顾」，
   把回顾正文塞进去，并把聊天窗弹出来给你看
   （还顺手填了一句"陪我回顾一下这阵子吧"当开场，方便你接着聊）
```

## 四、AI 是怎么被"教"着写的（prompt 的核心）

回顾的质量全靠那段 prompt 引导。它强制 AI 同时交付**两样价值，缺一不可**（这是这个功能的设计灵魂）：

1. **情感陪伴**：让你感到这段时间一直被陪着、被懂、被关心——温暖、有人味，但不肉麻。
2. **工作提效**：基于你的活动规律，帮你看清时间花在哪、有没有反复卡住 / 一直没收尾的事，并**温柔地给 1-3 条能真正减负的小建议**。

> 文档里写得很直白：第 2 点"是你一直陪着他、观察他的意义所在"——也就是说，**采集你的活动数据不是为了监控，而是为了能在这里实实在在帮到你**。这是收集数据的正当理由，也是底线。

prompt 还做了几件事：
- **语气可选**：开头注入你在设置里选的风格（温暖陪伴 / 简洁干练 / 俏皮活泼 / 鼓励打气 / 犀利点醒，来自 `BriefingStyle`），只改"怎么说"，不改"要说有用的东西"这条底线。
- **反流水账**：明确要求"别逐日复述数据，要提炼出『规律』和『故事感』"，篇幅约 350-600 字。
- **跟随界面语言**：你把界面切英文，回顾也用英文写。
- **软化措辞**：提到"没收尾的事"时要求用很软的语气、绝不武断断定，本质是帮你而不是打击你。

## 五、自动触发 vs 手动触发

你点的是**手动**，但这个功能平时也会**自动**冒出来。两者走同一套生成逻辑，区别只在"什么时候触发"和"看多少天"：

| | 触发条件 | 回看天数 | 数据门槛 | 标题 |
|---|---|---|---|---|
| **周报**（自动） | 距上次回顾 ≥ 7 天后的首次启动 | 7 天 | ≥ 3 天日报 | 🌱 这阵子的回顾 |
| **里程碑**（自动） | 认识满 30 / 100 / 365 天 | 30 天 | ≥ 1 天即可（重在纪念感） | 🎉 我们认识 N 天啦 |
| **手动**（你点的） | 随时点「立即生成」 | 14 天 | ≥ 3 天日报 | 🌱 这阵子的回顾 |

几个贴心细节：
- **里程碑优先于周报**：同一次启动如果两个都满足，先庆祝里程碑，不会一次弹两份。
- **不依赖你周日开机**：周报是"距上次≥7天就补做"，而不是"必须周日触发"，所以不会因为你哪天没开 app 就漏掉。
- **认识天数**：第一次启动那天会被记下来（`companionFirstDay`），里程碑就是从那天算起的。
- **防重入**：正在生成时再点不会重复跑（`isGenerating` 守卫）。

| 文件 | 角色 |

| `Sources/ChatViewModel.swift` | `streamOneShotAsk()` 调 AI、`createBriefingConversation()` 出对话 |

## 七、目前的局限（诚实说明）

- **要先攒够日报**：手动至少需要 3 天的日报存档，否则会提示"攒得还不够"。日报靠你每天打开 app 时自动生成，长期不开机就没料可回顾。
- **只看提炼后的要点，不看全文**：为了省 AI 的容量、也为了快，回顾读的是每天压缩好的要点（app / 文档 / 话题数），不是你那天的完整记录。所以它擅长说"规律和故事感"，不擅长复述某天的具体细节。
- **"意图感知"默认可能没开**：窗口标题、碰过的文档这些更细的料，需要你在设置 / 新手引导里开启「意图感知」（涉及辅助功能权限）才有。没开的话，回顾主要基于 app 使用时长和你问 AI 的话题。
- **AI 写的，偶尔会发挥**：回顾是 AI 基于数据"提炼"出来的，可能有少量主观推断，仅作参考、别太当真。
